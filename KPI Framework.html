KPI Framework

Структура построения и типологии показателей эффективности

// Базовые определения
KPI
Показатели для измерения и корректировки эффективности процессов. Не самоцель — инструмент управления.
Процесс
Циклично повторяемые действия для достижения определённой цели. KPI существует только если есть повторяемость.
// Три типа KPI
1
Работоспособность
KPI минимального порога
Циклично повторяемые действия для достижения минимально требуемых значений.
Логика: ≥ MIN
Вопрос: процесс вообще
работает?
Пример: менеджер делает 1 звонок в день → продаж нет. KPI = минимум 50 звонков/день чтобы процесс "завёлся".
2
Эффективность
KPI допустимого максимума
Циклично повторяемые действия для достижения максимально возможных значений, при которых процесс остаётся эффективным. При превышении этих значений процесс ломается или перестаёт быть эффективным.
Логика: ≤ MAX
Вопрос: процесс не
разрушает себя?
Пример: машина едет 150 км/ч → убивает двигатель. KPI = не выше 100 км/ч. Цель достигается, ресурс сохраняется.
3
Норматив
KPI целевого значения
Производное Типов 1 и 2. Когда определён минимальный порог работоспособности и максимальный порог эффективности — KPI превращается из граничного условия в точное целевое значение внутри допустимого диапазона.
Логика: MIN ≤ X ≤ MAX
Вопрос: какое точное
значение оптимально?
Пример: звонков ≥ 50, конверсия не падает при > 80 → целевой KPI = 65 звонков/день как оптимальная точка.

// Алгоритм определения KPI для процесса
1
Визуализировать и выписать все процессы
Собрать полный список циклично повторяемых действий. Без этого шага невозможно определить что измерять.
Map of processes
2
Этот процесс вообще нужен?
Прежде чем измерять — убедиться что процесс создаёт ценность. Процесс может быть работоспособным и эффективным, но не тем. Если процесс не нужен — его устранить, а не оптимизировать.
Validate purpose
3
Процесс работоспособен?
Даёт ли процесс хоть какой-то результат при текущей интенсивности? Если нет — процесс не включился. Нужно определить минимальный порог запуска.
→ Тип 1 KPI (MIN)
4
Есть ли данные для определения MIN?
Если процесс новый и исторических данных нет — минимальный порог нельзя взять из воздуха. Запустить эксперимент: поставить гипотезу, собрать первые данные, скорректировать MIN по факту. Не ставить пороговые значения без измерений.
Experiment → Baseline
5
Процесс эффективен?
Можно ли получить тот же результат дешевле — по времени или деньгам — без потери качества? Если процесс перегружен или избыточен — определить потолок после которого начинается деградация.
→ Тип 2 KPI (MAX)
6
Известны MIN и MAX?
Если оба порога определены — рассчитать оптимальную рабочую точку внутри допустимого диапазона. KPI превращается из граничного условия в точное целевое значение для ежедневного управления.
→ Тип 3 KPI (TARGET)
// Методологическое замечание
Тип 3 — производный, не самостоятельный. Он не существует без предварительного определения Типов 1 и 2. Попытка поставить целевое значение без понимания floor и ceiling — это либо случайное число, либо экспертная интуиция. Оба варианта нестабильны при изменении условий.

Последовательность обязательна: работоспособность → эффективность → норматив. Оптимизировать сломанный процесс бессмысленно.
