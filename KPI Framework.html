KPI FRAMEWORK
Structure, typology and construction of performance indicators
Basic Definitions
KPI
Indicators for measuring and adjusting process performance. Not an end in itself — a management tool.
Process
Cyclically repeated actions aimed at achieving a specific goal. A KPI only exists where there is repetition.
Three Types of KPI
1 Operability
Minimum Threshold KPI
Cyclically repeated actions to achieve the minimally required values.
Logic: ≥ MIN
Question: does the process
work at all?
Example: a manager makes 1 call per day → no sales. KPI = minimum 50 calls/day for the process to activate.
2 Efficiency
Maximum Threshold KPI
Cyclically repeated actions to achieve the maximum values at which the process remains effective. Exceeding these values breaks or degrades the process.
Logic: ≤ MAX
Question: does the process
destroy itself?
Example: a car drives at 150 km/h → engine damage. KPI = no more than 100 km/h. Goal achieved, resource preserved.
3 Standard
Target Value KPI
Derived from Types 1 and 2. Once the minimum operability threshold and the maximum efficiency ceiling are defined — the KPI becomes a precise target value within the acceptable range.
Logic: MIN ≤ X ≤ MAX
Question: what exact value
is optimal?
Example: calls ≥ 50, conversion doesn't drop above 80 → target KPI = 65 calls/day as the optimal point.

Algorithm for Defining KPI for a Process
1
Visualize and list all processes
Collect a complete list of cyclically repeated actions. Without this step it is impossible to determine what to measure.
Map of processes
2
Is this process even necessary?
Before measuring — verify that the process creates value. A process can be operable and efficient, but unnecessary. If the process is not needed — eliminate it, don't optimize it.
Validate purpose
3
Is the process operable?
Does the process produce any result at the current intensity? If not — the process has not activated. You need to define the minimum launch threshold.
→ Type 1 KPI (MIN)
4
Is there data to define MIN?
If the process is new and no historical data exists — the minimum threshold cannot be pulled from thin air. Run an experiment: set a hypothesis, collect initial data, adjust MIN based on actuals. Never set threshold values without measurement.
Experiment → Baseline
5
Is the process efficient?
Can the same result be achieved cheaper — in time or money — without quality loss? If the process is overloaded or excessive — define the ceiling beyond which degradation begins.
→ Type 2 KPI (MAX)
6
Are both MIN and MAX known?
If both thresholds are defined — calculate the optimal operating point within the acceptable range. The KPI transforms from a boundary condition into a precise target value for daily management.
→ Type 3 KPI (TARGET)
Methodological Note
Type 3 is derived, not standalone. It cannot exist without first defining Types 1 and 2. Attempting to set a target value without understanding the floor and ceiling is either a random number or expert intuition. Both are unstable when conditions change.
The sequence is mandatory: operability → efficiency → standard. Optimizing a broken process is pointless.
