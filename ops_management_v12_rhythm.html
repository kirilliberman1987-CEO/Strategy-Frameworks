Фундамент системы: KPI + OKR
Чтобы сравнивать план и факт на каждом уровне — нужно заранее определить целевые показатели. KPI фиксирует что измеряем. OKR задаёт куда идём и через какие результаты. Без этого нет плановых значений — нечего сравнивать.
KPI — что измеряем OKR — цели и ключевые результаты Planned vs Actual — на каждом уровне
Применяется во всех трёх блоках — Блок 1, Блок 2, Блок 3
🔄
Операционный ритм и стандартизация
— единые правила для каждого уровня управления
Применяется во всех трёх блоках
Блок 1
Daily · Weekly
Ритм: ежедневный дашборд + еженедельный митинг COO с Head of
Формат: 30–45 мин, фокус на одном буксующем показателе
Стандарт: дашборд открыт, план vs факт на экране, action items фиксируются письменно
Выход: конкретное решение + ответственный + дедлайн
Блок 2
Monthly
Ритм: фиксированный день месяца, одна и та же повестка каждый раз
Формат: 60–90 мин, COO + CEO, дашборд + отчёт подготовлены заранее
Стандарт: единый шаблон отчёта — метрика / план / факт / дельта / причина / action
Выход: обновлённый action plan на следующий месяц
Блок 3
Quarterly
Ритм: последняя неделя квартала, полный стратегический разбор
Формат: 2–3 часа, COO + CEO, данные Блока 2 × 3 месяца на входе
Стандарт: квартальный отчёт — тренды, паттерны, выводы, корректировка OKR
Выход: скорректированная стратегия + KPI-план Блока 2 на следующий квартал
